## 5. Issues and Recommendations

### 5.1 Line-number / location mismatch (high impact)

- **Symptom:** One issue ("OPENAI_RATE_LIMIT_MAX_RETRIES=0" / `||` vs `??`) never closed: fixer pointed at 287–300 (correct), verifier at "line 248".
- **Recommendation:** When attaching "current code" to analysis or verification, pass **current line numbers** or a stable identifier (e.g. function + snippet hash). Optionally detect when the cited line in the comment no longer matches the code (e.g. line 248 is now a comment) and treat as STALE or re-anchor the issue.

### 5.2 Final audit false positives (medium) — **Implemented**

- **Symptom:** Audit listed package.json and embedding.ts items as "need more work" when the audit text itself said "no change needed" or "addressed by comment".
- **Fix:** Post-process in `src/workflow/analysis.ts`: `isAuditNoActionNeeded(explanation)` matches phrases like "no change needed", "correct state", "cautionary not prescriptive". Those issues are marked verified and do not re-enter the fix loop.

### 5.3 Quota handling (low)

- **Symptom:** Codex hit quota on first use; rotation to LLM API was correct.
- **Recommendation:** No code change required; optionally back off or delay before first Codex call if quota errors are common in this environment.
